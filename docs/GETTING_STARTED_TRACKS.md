# Getting Started Tracks

Choose the track that matches your immediate goal.

- [Quickstart Track](GETTING_STARTED_QUICKSTART.md): fastest path from clone to running `boomhauer` app.
- [API-First Track](GETTING_STARTED_API_FIRST.md): JSON API setup, schema/auth contracts, and OpenAPI docs.
- [HTML-First Track](GETTING_STARTED_HTML_FIRST.md): server-rendered EOC templates, layouts, forms, and validation rendering.
- [Data-Layer Track](GETTING_STARTED_DATA_LAYER.md): SQL builder + PostgreSQL default path + optional MSSQL + migrations + standalone `ArlenData` usage.

If you are unsure, start with the quickstart track and then continue with:

- `docs/APP_AUTHORING_GUIDE.md` for day-to-day route/controller/middleware work
- `docs/CONFIGURATION_REFERENCE.md` for config keys and inspection
- `docs/MODULES.md` for first-party modules
- `docs/LITE_MODE_GUIDE.md` if you want the smaller single-file app shape
